# Security Policy

We really care about keeping student data and academic materials safe. If you're poking around the code and find a vulnerability, we would absolutely love your help in fixing it.

## Supported Versions

Right now, we only push security updates to the latest version on the `main` branch. Since we are a small team, we don't maintain older releases. If you find a bug, assume it applies to the current live code.

## How to Report a Vulnerability

If you find something serious—like a way to bypass authentication, access private files, or inject malicious scripts—**please don't open a public issue on GitHub**. We don't want someone else taking advantage of the exploit before we have a chance to patch it.

Instead, please email me directly at:
**krishnabobade1313@gmail.com**

Try to include:
- A quick summary of what the vulnerability is.
- Step-by-step instructions on how we can reproduce it locally.
- (Optional) Ideas on how you think we should fix it.

I usually check my email every day, so I'll try to get back to you within 24 hours to confirm we got your report. From there, we can work together to get a fix pushed out. 

Thanks for looking out for the community!
