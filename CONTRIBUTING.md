# Contributing to the Study Repository

Hey! First off, thanks for even opening this file. We honestly appreciate anyone willing to spend their free time helping improve this project. 

Whether you're fixing a typo, squashing a bug, or proposing a brand new feature, we're happy to have you here. Don't worry if you're new to open source or React—we'll help you figure it out.

## Found a bug?
If something is broken, please let us know! Just open a new issue. 
When you do, try to give us as much info as you can:
- What exactly did you click or do before it broke?
- What did you expect to happen, and what actually happened?
- If you can grab a screenshot, that helps a ton.
- Mention what browser (Chrome, Safari, etc.) and OS you're using.

## Got an idea?
We're always looking to make the platform better for students. If you have an idea for a cool feature, open an issue and tell us:
- What problem it solves for you.
- How you think it should work.

## How to make changes (Pull Requests)

Ready to write some code? Here's our basic workflow:

1. **Fork the repo** and clone it to your machine.
2. **Create a new branch** off `main` for your work (e.g., `git checkout -b fix-login-button`).
3. **Write your code.** Try to stick to the style we already have (Tailwind classes, functional React components). If you're adding something that needs a loading state, try to use our existing Skeleton components so it doesn't look jarring.
4. **Test it locally** to make sure you didn't break anything else by accident.
5. **Open a Pull Request!** Write a quick description of what you changed so we know what to look for.

## Security issues
If you happen to find a serious security flaw (like a way to bypass login or access someone else's data), please **do not open a public GitHub issue**. We don't want people exploiting it before we can fix it. Instead, just email krishnabobade1313@gmail.com directly, and we'll patch it ASAP.

Thanks again for helping us build something useful!
