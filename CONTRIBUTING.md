# Contributing to MIT-WPU Study Repository

First off, thank you for considering contributing to the Study Repository! It's people like you that make this a great academic resource for everyone.

## Code of Conduct
By participating in this project, you agree to abide by our code of conduct. Please be respectful and professional in all interactions.

## How Can I Contribute?

### Reporting Bugs
If you find a bug, please create a new issue. Include as much detail as possible, such as:
- Steps to reproduce the bug
- Expected behavior vs actual behavior
- Screenshots if applicable
- Your browser and OS version

### Suggesting Enhancements
We are always looking for ways to improve! If you have an idea, please open an issue and describe:
- The problem you're trying to solve
- Your proposed solution
- Why this would be beneficial for the community

### Submitting Pull Requests
1. Fork the repo and create your branch from `main`.
2. Ensure your code follows the existing style and linting rules.
3. If you've added code that should be tested, add tests.
4. Update the documentation if needed.
5. Issue that pull request!

## Style Guidelines
- Use functional components with hooks.
- Follow the established design system (Tailwind + CSS Variables).
- Write descriptive commit messages.
- Ensure all new features have appropriate loading states (Skeletons).

## Security
If you discover a security vulnerability, please do NOT open a public issue. Instead, email [krishna.bobade@mitwpu.edu.in](mailto:krishna.bobade@mitwpu.edu.in) directly.

Thank you!
